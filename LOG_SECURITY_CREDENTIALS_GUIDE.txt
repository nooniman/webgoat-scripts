====================================
FINDING ADMIN CREDENTIALS IN LOGS
====================================

CHALLENGE: Find secret admin credentials in WebGoat server logs

CLUES:
1. Credentials provided at "boot-up of the server"
2. They are in the "application log"
3. They are "protected" (encoded somehow)
4. We need to decode them
5. Use them to login as Admin

====================================
ATTACK STRATEGY
====================================

STEP 1: CHECK WEBWOLF FOR LOGS
--------------------------------------
WebWolf (http://192.168.254.112:8002) often captures logs and requests.
Check:
- /logs
- /files
- /requests
- Any log viewer endpoints

STEP 2: CHECK WEBGOAT ENDPOINTS
--------------------------------------
Try common log endpoints:
- /WebGoat/logs
- /WebGoat/app.log
- /WebGoat/application.log
- /WebGoat/server.log
- /WebGoat/boot.log
- /WebGoat/startup.log
- /WebGoat/debug.log

STEP 3: INSPECT BROWSER CONSOLE
--------------------------------------
The lesson page might reveal log information:
- Open browser console (F12)
- Check Network tab for log-related responses
- Look for any encoded strings in page source

STEP 4: COMMON ENCODING SCHEMES
--------------------------------------
"Protected" credentials could be encoded as:
- Base64 (most common)
- Hex
- URL encoding
- ROT13
- MD5/SHA hashes (unlikely for credentials)
- Caesar cipher

====================================
COMMON LOG PATTERNS TO LOOK FOR
====================================

1. Startup messages:
   "Admin password: XXXXX"
   "Default credentials: admin/XXXXX"
   "Bootstrap user created: admin:XXXXX"

2. Base64 encoded:
   Looks like: YWRtaW46cGFzc3dvcmQxMjM=
   Contains: A-Z, a-z, 0-9, +, /, =

3. Hex encoded:
   Looks like: 61646d696e3a70617373776f726431323334
   Contains: 0-9, a-f

4. Spring Boot common patterns:
   "Using generated security password: XXXXX"
   "Admin credentials: XXXXX"

====================================
WHERE TO SUBMIT CREDENTIALS
====================================

Once found, try:
1. WebGoat login page
2. The lesson's login form
3. Specific endpoint: /WebGoat/LogSecurity/login
4. POST with: username=admin&password=DECODED_PASSWORD
